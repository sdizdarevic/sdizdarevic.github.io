Sparrowboard 1450 Numato getting started

Sparrowboard1450 is a compact and simple development board for learning 18 series Microchip PIC microcontrollers. Numato Lab is making really good products which are really simple to use and learn electronics and are by my opinion designed for schools and universities to teach students. With this easy-to-use products you spend your time only on coding and not on design.

The Sparrowboard1450 is a general purpose processor module using the PIC18F14K50, so it includes a USB interface with Microchip’s HID Boot-Loader already programmed in. The Sparrowboard1450 has a PIC18F14K50 USB PIC populated with built in (Microchip) bootloader. This single piece of hardware is all you need to get your first program up running. No programmer or other hardware tool is required.

The board features 12MIPs processor performance, and button selectable reset to Bootloader or Application.

Compiling programs to run with the boot-loader
The boot loader occupies program memory from 0x0000 to 0x0FFF. Your programs must therefore be linked to run at 0x1000. The reset and interrupt vectors which are normally at 0x0000, 0x0008 and 0x0018 are re-mapped to 0x1000, 0x1008 and 0x1018. You must also include the rm18F14K50.lkr linker script in your project (if you dont include the linker file, the bootloader will be overwritten). The linker script tells the linker how to re-locate your program to the correct place. If you dont want to mess with this, you could just download the getting started file below,extract it, and compile it with free microchip compiler that comes with great-looking ide based on Netbeans IDE.

If you are interested in how vector remapping works, you could keep reading.

VECTOR REMAPPING
On PIC18 devices, addresses 0x00, 0x08, and 0x18 are used for
the reset, high priority interrupt, and low priority interrupt
vectors. However, the current Microchip USB bootloader occupies
addresses 0x00-0xFFF. Therefore the bootloader code remaps these
vectors to new locations. So you just simply add these three lines in your main.c file:

#define REMAPPED_RESET_VECTOR_ADDRESS 0x1000
#define REMAPPED_HIGH_INTERRUPT_VECTOR_ADDRESS 0x1008
#define REMAPPED_LOW_INTERRUPT_VECTOR_ADDRESS 0x1018

What we are going to do is the first “hello world” program for this microcontroller that
will blink LED4. (LEDS are connected to Microcontroller GPIOs in the following order:
LED1 -> RC7
LED2 -> RC6
LED3 -> RC5
LED4 -> RC4

SO the code now looks like this:

#define LED4 LATCbits.LATC4 // LED4 -> RC4

void main(void)
{
ANSEL = 0; // all pins to digital mode
ANSELH = 0;

TRISB = 0xff; // all pins to input;
TRISC = 0xef; // Port C4 (RC4 -> Led4) to output mode
T3CON = 0xb1; // 16 bit, 8:1 prescale, running

while(1) {
LED4 = 0;
delay_mS(400);
LED4 = 1;
delay_mS(300);

}
}

The delay_mS() if just a delay function that takes integer as argument
and keeps LED on or off for entered mS.

How to load firmware

Sparrowboard1450 is preprogrammed with Microchip compatible bootloader. When plugged to USB port,
the board will start executing the application firmware normally. To download firmware, run Microchip
Bootloader application (available as part of Microchip Application Libraries) and press the reset switch
while ‘SW’ switch is being pressed. While coming out of reset, the on board bootloader will sample the
state of RB4 GPIO and will go to download mode if the ‘SW’ switch is pressed.
If the board is entered bootloader mode, you can see the buttons on the Microchip bootloader
application becomes active. At this point, your application firmware can be downloaded to the board.

If you want more info about this board, you can go to: http://numato.com/sparrowboard-1450-user-guide
